# Installed ForgeKit Modules

This file lists the ForgeKit modules installed in this project together
with their licenses, versions, and source repositories. The table is
maintained by the `Module_Installer` component of the MCP server: when
you unzip a paid module into `addons/`, the installer rewrites the
corresponding row. Do not edit this file by hand unless you are also
updating the matching `module.manifest.tres`.

## ForgeKit modules

| Module          | License                                                                      | Version | Repository                                                                                      |
| --------------- | ---------------------------------------------------------------------------- | ------- | ----------------------------------------------------------------------------------------------- |
| `forgekit_core` | [MIT](./LICENSE)                                                             | 0.7.0   | https://github.com/ForgeKitStudio/forgekit-core                                                 |
| `forgekit_rpg`  | Commercial EULA — see the product page on [itch.io](https://forgekitstudio.itch.io/forgekit-rpg) / [Gumroad](https://forgekitstudio.gumroad.com/l/forgekit-rpg) | 0.7.0   | https://github.com/ForgeKitStudio/forgekit-rpg _(private; purchase required to gain access)_    |

The `forgekit_rpg` entry describes the paid module distributed from
`ForgeKitStudio/forgekit-rpg`. The source repository is private; buyers
receive the signed ZIP through their itch.io or Gumroad dashboard and
install it under `addons/forgekit_rpg/`. The `Module_Installer`
populates the License, Version, and Repository columns from the
module's `module.manifest.tres` (fields `license_id`, `version`,
`source_repo`) once the extraction finishes. The row above reflects
the latest tag shipped at the time this notice was written; it is
rewritten in place when a newer module build is installed.

### `@forgekit/core-mcp` npm dependencies

The Node.js MCP server published from `mcp-server/` declares a small
set of dependencies. Their licenses and roles are listed below; the
canonical license texts live inside each package under
`node_modules/<package>/LICENSE` after running `npm ci`.

| Package         | Type       | License    | Role                                                                          | Upstream                                      |
| --------------- | ---------- | ---------- | ----------------------------------------------------------------------------- | --------------------------------------------- |
| `ws`            | runtime    | MIT        | WebSocket transport used by the editor-plugin channel.                        | https://github.com/websockets/ws              |
| `typescript`    | dev        | Apache-2.0 | Compiler producing the published `dist/` bundle.                              | https://github.com/microsoft/TypeScript       |
| `vitest`        | dev        | MIT        | Test runner for the property-based and unit test suites.                      | https://github.com/vitest-dev/vitest          |
| `fast-check`    | dev        | MIT        | Property-based testing library used by every `property_*.test.ts` suite.      | https://github.com/dubzzz/fast-check          |
| `@types/node`   | dev        | MIT        | TypeScript type definitions for Node.js.                                      | https://github.com/DefinitelyTyped/DefinitelyTyped |
| `@types/ws`     | dev        | MIT        | TypeScript type definitions for the `ws` WebSocket library.                   | https://github.com/DefinitelyTyped/DefinitelyTyped |

Dev-time dependencies (`typescript`, `vitest`, `fast-check`, the two
`@types/*` packages) are excluded from the published tarball but are
required to build and test the server locally or in CI.
